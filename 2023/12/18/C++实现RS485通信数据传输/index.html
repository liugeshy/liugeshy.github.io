C++实现RS485通信控制pelco-d协议云台运动

shy · 2023-12-18 · 次阅读

字数统计: 1.7k字   |   阅读时长: 7分

  最近实验室买了个云台，于是想要利用之前写的一个CUDA加速KCF的程序进行目标跟踪，同时利用PID控制云台，使云台能够实时跟踪目标运动，也就是目标框总是处于摄像头画面的中心。
  首先要解决的问题就是如何实现上位机与云台之间的通信，串口利用的是RS485通信协议，关于RS485通信协议详解可以参见：https://blog.csdn.net/weixin_46251230/article/details/126684223
  云台的控制是基于PELCO-D协议，具体协议指南参见：https://blog.csdn.net/u014436243/article/details/107639901
  在python上解决串口通信的问题很简单，可以直接调用serial库，而在C++中就有些困难，尝试了很多办法，也有使用开源的C++编写的serial.h库，但是没有成功（可能是发送的数据类型的问题，而且当时编译的时候没有导入pthread库应该也是失败的原因，感兴趣的同学可以尝试一下），最终利用boost.asio库实现了串口通信。boost.asio库安装
  一个简单的示例代码来检验boost.asio库是否安装成功

#include <iostream>
#include <boost/asio.hpp>
int main()
{
  boost::asio::io_context io;
  boost::asio::steady_timer t(io, boost::asio::chrono::seconds(5));
  t.wait();
  std::cout << "Hello, world!" << std::endl;
  return 0;
}

确定自己的库的安装位置，这里我的是/usr/include，然后用下面的指令编译
g++ example1.cpp -o a.out -L /usr/include -lboost_system -lboost_thread -lpthread
  确认安装成功后下一步就要正式开始串口通信了，首先要打开串口并设置串口参数

boost::asio::io_service io;
boost::asio::serial_port port(io,"/dev/ttyUSB0");
port.open("/dev/ttyUSB0",m_ec);
port.set_option(boost::asio::serial_port::baud_rate(9600));
port.set_option(boost::asio::serial_port::flow_control(boost::asio::serial_port::flow_control::none));
port.set_option(boost::asio::serial_port::parity(boost::asio::serial_port::parity::none));
port.set_option(boost::asio::serial_port::stop_bits(boost::asio::serial_port::stop_bits::one));
port.set_option(boost::asio::serial_port::character_size(8));

  peclo-d协议指令格式为：FF Addr Command1 Command2 Data1 Data2 CS，其中，CS=(Addr+Command1+Command2+Data1+Data2)&0xFF
  我们把要传输的数据定义为一个uint8_t型的数组，然后分别对数组的元素进行赋值，之后通过boost::asio::write命令写入串口，关于如何将传输的数据定义为数组可以参考：https://blog.csdn.net/GUA8122HOU/article/details/120565812

  下面是将数据写入串口的示例代码，简单的实现了让云台向上转动一段时间后停止

#include <iostream>
#include <boost/asio.hpp>
#include <fstream>
#include <sstream>
#include <algorithm>
#include <ctime>
#include <chrono>
#include <vector>
#include <iomanip>
#include <cmath>
#include <time.h>
#include <unistd.h>
using namespace std;
boost::system::error_code m_ec;
boost::asio::io_service io;
boost::asio::serial_port port(io,"/dev/ttyUSB0");
uint16_t  ADD8_Check(uint8_t *num_Data,uint16_t num_Size)
{
    uint16_t i;
    uint16_t data_sum=0;
    for(i=1; i<num_Size; i++){
        data_sum += num_Data[i];
    }
    data_sum = (data_sum%256)&0xFF;    
    return data_sum;
}

int main()
{



        port.open("/dev/ttyUSB0",m_ec);
        port.set_option(boost::asio::serial_port::baud_rate(9600));
        port.set_option(boost::asio::serial_port::flow_control(boost::asio::serial_port::flow_control::none));
        port.set_option(boost::asio::serial_port::parity(boost::asio::serial_port::parity::none));
        port.set_option(boost::asio::serial_port::stop_bits(boost::asio::serial_port::stop_bits::one));
        port.set_option(boost::asio::serial_port::character_size(8));

        if (port.is_open()) 
        {
        std::cout << "Serial port is open." << std::endl;
        } 
        else 
        {
        std::cerr << "Failed to open serial port." << std::endl;
        return 1;
        }

        uint8_t PTZ_TX_BUF[7]={};
        uint8_t stop[7]={};


    PTZ_TX_BUF[0]=0xFF;
    PTZ_TX_BUF[1]=0x01;     //PTZ地址码
    PTZ_TX_BUF[2]=0x00;                 //command1
    PTZ_TX_BUF[3]=0x08;                    //command2
    PTZ_TX_BUF[4]=0x00; 
    PTZ_TX_BUF[5]=0x63; 
    uint8_t check_num= ADD8_Check(PTZ_TX_BUF,6);
    PTZ_TX_BUF[6]=check_num; 
        stop[0]=0xFF;
    stop[1]=0x01;     //PTZ地址码
    stop[2]=0x00;                 //command1
    stop[3]=0x00;                    //command2
    stop[4]=0x00; 
    stop[5]=0x00; 
    uint8_t check_num2= ADD8_Check(stop,6);
    stop[6]=check_num2; 

        boost::asio::write(port,boost::asio::buffer(PTZ_TX_BUF),m_ec);
        sleep(1);
        boost::asio::write(port,boost::asio::buffer(stop),m_ec);
        sleep(0.2);

    return 0;
}

输入下面的指令运行 test1:g++ test1.cpp -o a.out -L /usr/include -lboost_system -lboost_thread -lpthread ，因为boost.asio库中可能会用到多线程操作，所以要链接thread库，不然会报错。
  实现了向云台写入数据，之后要做的就是如何从云台读取视频，本次实验中我是直接利用opencv中的videocapture直接读取rtsp视频流，测试代码如下：

#include <iostream>
#include "opencv2/opencv.hpp"
#include <opencv2/core/core.hpp>
#include <opencv2/highgui/highgui.hpp>
#include <opencv2/video.hpp>
#include <opencv2/imgproc/imgproc.hpp>
#include <opencv2/videoio/videoio.hpp>
using namespace cv;

int main()
{

    VideoCapture capture;
    capture.open("rtsp://192.168.1.108:554/id=1&type=0");

    //创建VideoCapture对象的另一种方式
    //VideoCapture capture("rtsp://admin:hikvision2021@192.168.1.64");


    Mat frame;

    while (1)
    {
        capture >> frame;

        imshow("test", frame);
        if(waitKey(30)>=0)

    }

  在终端中输入g++ track.cpp -o run `pkg-config --cflags --libs opencv`
  一开始直接运行的时候并没有加上`pkg-config --cflags --libs opencv`命令，导致报错undefined reference to cv::VideoCapture::VideoCapture()，明明头文件中导入了opencv库却报错说没有找到videocapture，究其原因，应该是opencv lib动态库没有添加完全，于是加上 `pkg-config --cflags --libs opencv`命令后顺利解决。
  测试的时候videocapture捕捉到的云台视频延迟还挺低，但是放到跟踪算法里，延迟就很高了，因为我后面要用PID控制，延迟这么高肯定是不行的，要想办法降低延迟。产生延迟的主要原因在于Opencv的videocapture库在开启后，会将照片数据不断存入缓冲区，而代码会将该缓冲区从时间最早的那张图片开始处理，如果代码处理速度小于存缓冲区的速度，则缓冲区的照片一直是满的，程序一直取的是缓冲区最早的那张照片，因此会出现几百毫秒的延迟。
  解决方案是新建一个线程，在该线程创建一个Vector < Mat >容器，并不停地将OpenCV缓冲区的照片以堆栈的形式存到该容器中，而主线程的处理程序以取栈顶的方式从容器中取出要处理的照片，因此，主程序将一直处理的是最新的照片
降延迟的方法主要参考：降低延迟
  好了，这下摄像头可以基本上实时采集画面了，呼，任务基本上要完成了，但是我在运行的时候又发现了另一个问题，就是程序运行的时候有时候会报错Assertion fctx->async_lock failed at libavcodec/pthread_frame.c:155,有时候就不会，或者反复运行几次之后就能正常运行，网上觉得说的比较有道理的就是可能是因为rtsp服务器还没有准备好连接，可以看这篇帖子的讨论：https://github.com/dactylroot/rtsp/issues/18,现在万事俱备，就只差我们最后的PID控制了。
